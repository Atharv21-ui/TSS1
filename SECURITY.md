## Reporting a Vulnerability

We take the security of the TSS Hardware platform seriously. If you believe you have discovered a security vulnerability in this repository, please report it to our security team immediately.

### How to Submit a Report

Please **do not** report security vulnerabilities through public GitHub issues. Instead, submit your finding via one of the following channels:

1. **Private Email**: Send an email to `security@tss-hardware.com` (or contact repository administrators directly).
2. **GitHub Private Vulnerability Reporting**: If enabled on this repository, submit directly via the **Security** tab under **Report a vulnerability**.

### Report Details

When reporting a vulnerability, please include as much of the following information as possible to help us understand and resolve the issue quickly:

- **Type of issue** (e.g., XSS, CSRF, Authentication bypass, API endpoint vulnerability).
- **Step-by-step instructions** to reproduce the issue.
- **Proof of Concept (PoC)** code or screenshots, if available.
- Impact assessment and potential risks.

## Response & Disclosure Timeline

- **Initial Acknowledgment**: Within 24–48 hours of receiving your report.
- **Assessment & Fix**: We aim to verify and release patches for confirmed vulnerabilities as quickly as possible.
- **Public Disclosure**: Coordinated disclosure will occur after a fix has been deployed and verified.
